# Customor QC App 项目简介

## 项目定位

`customor-qc-app` 是一个面向电商客服团队的质检与训练平台，用来把客服聊天记录转成可管理、可追踪、可训练的质量提升闭环。

它的核心目标是：
- 持续发现服务问题（自动质检 + 标签体系）
- 快速定位问题来源（客服、场景、满意度、对话明细）
- 形成改进动作（训练任务、评分反馈、复盘）

## 核心功能

1. 多角色权限与账号体系
- 支持管理员 / 主管 / 客服角色
- 支持外部客服账号绑定站内用户

2. 对话数据导入与批次管理
- 支持导入 AI 对话分析 JSON（txt）数据
- 生成批次并可下钻到具体对话详情

3. AI 质检与自动化触发
- 支持手动触发 AI 质检
- 支持每小时自动检查并对符合条件的对话自动入队质检
- 导入后可立即触发自动质检检查

4. 未绑定账号检测与提醒
- 导入后自动识别未绑定客服账号
- 通过飞书 webhook 发送提醒，辅助运维快速补齐绑定

5. 标签体系与质量分析
- 支持标签管理、分类管理、标签合并、手动标记
- 支持标签报表与多维统计

6. 客服训练闭环
- 主管可针对问题对话创建训练任务
- 客服提交改写练习，主管评分并反馈

7. 报表与管理视图
- 客服分析报表（按日/周/月，按场景/满意度下钻）
- 对话列表、详情页、任务页、配置页等完整后台界面

## 典型使用流程

1. 管理员/主管导入聊天数据
2. 系统识别未绑定账号并通知
3. 自动或手动触发 AI 质检
4. 在报表或标签视图中定位问题
5. 创建训练任务并跟进改进效果

## 技术实现概览

- 后端：FastAPI + SQLModel
- 数据库：PostgreSQL
- 前端：Jinja2 Templates + CSS
- 运行方式：Docker Compose（app/db/worker/cron）
- 核心目录：
  - `app/main.py`：主要路由与页面
  - `app/models.py`：数据模型
  - `app/importers/`：导入逻辑
  - `app/qc_checker.py`：自动质检筛选逻辑
  - `app/worker.py`：后台任务执行
  - `app/cron_runner.py`：定时导入与定时检查

## 快速启动

```bash
docker compose up --build
```

启动后访问：
- `http://localhost:8000`

## 适用对象

- 客服团队负责人（看质量趋势与管理效率）
- 质检/培训团队（发现问题并组织训练）
- 业务运营团队（从对话中沉淀服务改进机会）
